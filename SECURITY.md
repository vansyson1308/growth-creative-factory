# Security Policy

## Supported Versions

This project is currently in active development and only the latest `main` branch is supported for security fixes.

## Reporting a Vulnerability

Please **do not** report security vulnerabilities in public GitHub issues.

Instead, email: **security@growth-creative-factory.dev** with:

- A clear description of the issue and impact.
- Reproduction steps or proof of concept (if available).
- Affected version/commit.
- Any suggested remediation.

We will acknowledge receipt within 3 business days and provide a remediation timeline once triaged.

## Secrets and Sensitive Data

When reporting a vulnerability, do not include:

- API keys/tokens (Anthropic, Meta, Google, etc.).
- OAuth client secrets.
- Service-account JSON files.
- Real customer data exports.

If needed, redact values and provide synthetic examples.
